# Contributing

## Scope

CredB Nexus is being rebuilt from a clean baseline. Contributions should stay aligned with the project direction and phase-based development plan.

## Working Rules

- keep changes focused
- avoid unrelated refactors
- use short-lived feature branches
- document architecture-impacting changes
- keep crawling separate from AI inference

## Branching

Use `main` as the stable branch.

Create short-lived working branches such as:

- `docs/*`
- `crawler/*`
- `backend/*`
- `frontend/*`
- `ai/*`
- `db/*`

## Pull Requests

Each pull request should include:

- what changed
- why it changed
- how it was validated
- whether documentation also changed

## Documentation

If repository structure, workflow, architecture, API shape, or deployment behavior changes, update the relevant docs in the repo and wiki.
